# Project Management

## Purpose
Track work for each phase, surface blockers, and keep documentation aligned with the roadmap.

## Getting Started
1. Use `roadmap/PROJECT-TRACKING.md` for milestone-level status.
2. Use PHASE-1-BACKLOG.md to capture stories, owners, and blockers for Phase 1.
3. Update these files as we complete tickets and shift phases.

## Collaboration
- Link deliverables to GitHub issues or backlog cards.
- Note dependencies, risks, and production readiness in the backlog entry.
